## Prototipado

En el desarrollo de software, un prototipo es un modelo de trabajo rudimentario de un producto o sistema de información, generalmente construido para Con fines demostrativos o como parte del proceso de desarrollo. En el modelo de prototipado del ciclo de vida del desarrollo de sistemas (SDLC), una versión básica del sistema se construye, se prueba y se vuelve a trabajar según sea necesario hasta que finalmente se obtenga un prototipo aceptable Logrado a partir del cual se puede desarrollar el sistema o producto completo.

La creación de prototipos se refiere a una etapa inicial de una versión de software en la que pueden ocurrir la evolución del desarrollo y las correcciones del producto Antes de que se inicie un lanzamiento más grande. A este tipo de actividades también se les puede llamar a veces fase beta o prueba beta. donde un proyecto inicial es evaluado por una clase más pequeña de usuarios antes del desarrollo completo.

La creación de prototipos, así como las pruebas de amplio espectro y las múltiples versiones de software, son parte de un proceso más detallado para Produciendo productos y servicios de software sofisticados. La idea esencial es que incluso cuando las funciones del código están completas en un proyecto, el software, que todavía está en desarrollo, puede tener muchos errores y problemas de usuario. Para muchos de estos para ser planchados Además, ayuda si el software está realmente en uso, pero los desarrolladores enfrentan el problema de lanzar un producto que los usuarios finales pueden ver. como esencialmente defectuoso. Se puede liberar el producto a una comunidad más pequeña o restringir su desarrollo en etapas. Una solución muy efectiva.

En algunos casos, la creación de prototipos puede involucrar a voluntarios, mientras que en otros casos, los clientes especiales u otros con estatus especial pueden Estar involucrado en la creación de prototipos. Los equipos de desarrollo y las compañías de tecnología generalmente obtendrán comentarios sobre un producto durante Prototipado para solucionar problemas antes de la distribución final.
