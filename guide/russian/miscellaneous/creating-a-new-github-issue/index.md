---
title: Creating a New GitHub Issue
localeTitle: Создание новой проблемы GitHub
---
Перед отправкой вопроса попробуйте найти [свой вопрос на GitHub](https://forum.freecodecamp.com/t/searching-for-existing-issues-in-github/18390)

Создание хорошей проблемы облегчит работу команды разработчиков и решает вашу проблему. Для этого выполните следующие действия:

1.  Перейдите на страницу « [Проблемы](https://github.com/FreeCodeCamp/FreeCodeCamp/issues) с [GitHub](https://github.com/FreeCodeCamp/FreeCodeCamp/issues) для FreeCodeCamp» и нажмите « `New Issue` .
    
2.  **Иметь полезное название**
    
    *   Напишите значащее название, описывающее проблему. Некоторые хорошие примеры: `Logging in from the News and Field Guide pages doesn't redirect properly (using e-mail)` и `Typo: "for" instead of "while" loop` ; плохие примеры включают `A bug, HELP!!!11` и `I found this bug in a Challenge` .
        
    *   Держите заголовок относительно коротким, описание для дополнительной информации. Одним из примеров является сокращение длинных имен вызовов, поэтому вместо того, чтобы писать `Test case bug in 'Challenge: Check Radio Buttons and Checkboxes by Default'` , вы можете захотеть записать `Test case bug in 'Radio Buttons' Challenge` .
        
3.  В теле **укажите ссылку** на страницу, на которой вы столкнулись с этой проблемой.
    
4.  **Опишите проблему** и **предоставьте шаги,** чтобы разработчик мог попытаться воспроизвести проблему. Включите операционную систему и версию браузера.
    
    *   Когда вы ссылаетесь на другие вопросы или тянете запросы, просто напишите #issue / pr-number.
5.  Вставить в любой соответствующий код, используя правильное [форматирование кода](https://forum.freecodecamp.com/t/markdown-code-formatting/18391)
    
6.  **Сделайте снимок экрана** и включите его в сообщение.
    
7.  Нажмите « `Submit New Issue` и все готово! Вы будете автоматически подписаны на уведомления о любых обновлениях или будущих комментариях.
