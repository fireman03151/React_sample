## Безопасный

SAFe означает «Scaled Agile Framework» и представляет собой гибкую платформу разработки программного обеспечения, предназначенную для крупномасштабного гибкого развития, включающего множество команд.

Основной веб-сайт ( [http://www.scaledagileframework.com/](http://www.scaledagileframework.com/) ) - свободно доступная онлайн-база знаний по внедрению SAFe.

## принципы

Существует девять основных принципов SAFe

1.  **Возьмите экономический взгляд** - оцените экономические издержки решений, событий и рисков
2.  **Применять системное мышление** - процессы развития - это взаимодействие между системами, используемыми рабочими, и, следовательно, необходимо учитывать прогресс в системном мышлении
3.  **Предположим, что изменчивость; сохранить варианты** - изменения требований, быть гибкими и использовать эмпирические данные для узкого внимания
4.  **Постройте постепенно с помощью быстрых интегрированных циклов обучения** - быстрые, инкрементные сборки позволяют быстро получать обратную связь, чтобы при необходимости изменить проект
5.  **Основные этапы объективной оценки рабочих систем** - объективная оценка предоставляет важную информацию, такую ​​как финансовые показатели и управление, как отзывы
6.  **Визуализировать и ограничивать WIP, уменьшать размер партии и управлять длинами очереди** - делать это для достижения непрерывного потока для быстрого перемещения и визуализации прогресса; WIP = незавершенное производство
7.  **Применить каденцию, синхронизировать с междоменным планированием** - каденция устанавливает даты, когда происходят определенные события (например, выпускаются еженедельно) и синхронизация гарантирует, что все имеют одинаковые цели
8.  **Разблокировать внутреннюю мотивацию работников знаний** - люди работают лучше, когда вы используете свои личные мотивы
9.  **Децентрализовать принятие решений** - позволяет ускорить действие, которое может быть не лучшим решением, но позволяет быстрее общаться между командами; централизованное принятие решений может потребоваться для принятия более стратегических или глобальных решений

## Конфигурации

Существует четыре варианта SAFe, которые различаются по сложности и потребностям вашего проекта:

1.  Существенный SAFe
2.  Портфолио SAFe
3.  Большое решение SAFe
4.  Полный SAFe
