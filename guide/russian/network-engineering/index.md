## Сетевая инженерия

**Пакет** представляет собой абстракцию передаваемых данных на _сетевом уровне_ модели OSI. Он состоит из фактических данных и других полей, необходимых для его посредничества от одного узла к другому узлу в сети. Поскольку передача всех данных по сети за один раз неэффективна, она разбивается на небольшие пакеты и затем передается.

Пакет выглядит следующим образом:

![пакет](https://www.computerhope.com/jargon/p/packet.jpg)

Сетевой слой разбивает данные на небольшие пакеты, каждый с уникальным идентификатором. Затем они передаются на транспортный уровень, который переносит их в пункт назначения через несколько промежуточных узлов, таких как маршрутизаторы и коммутаторы, с использованием _протокола управления передачей_ (TCP) и _протокола Интернета_ (IP). TCP имеет дело с фактической маршрутизацией, в то время как IP помогает в логической адресации устройств (IP-адрес), чтобы отслеживать, какой узел отправляет пакет рядом.

Как только пакеты достигают пункта назначения, подтверждение отправляется обратно. Если по какой-либо причине пакет удаляется, его можно повторно запросить с исходного узла.

Сетевые инженеры отвечают за внедрение, поддержку, поддержку, разработку и, в некоторых случаях, разработку сетей связи в организации или между организациями.

Их цель - обеспечить целостность сетевой инфраструктуры высокой доступности, чтобы обеспечить максимальную производительность для своих пользователей. Пользователи могут быть сотрудниками, клиентами, клиентами и поставщиками. Сети могут включать:

компьютер; голос; брандмауэр. Сетевые инженеры могут работать внутри организации в составе группы поддержки ИТ-подразделения организации или извне в составе аутсорсинговой консалтинговой фирмы по ИТ-сетям, работающей с несколькими клиентами.
