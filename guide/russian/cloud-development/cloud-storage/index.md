## ХРАНЕНИЕ КАК СЕРВИС (StAAS)

Когда мы используем хранилище облаков, это StAAS. В этом облачная система имеет хранилище, и пользователь использует это хранилище в своей системе и может использовать это для хранения данных. Облачная система должна иметь возможность масштабирования хранилища. Мы можем интегрировать все хранилища. Например, если у нас есть несколько ручек, мы можем интегрировать все хранилища для накопителей в одном.

В основном существуют два типа хранилищ

1.  объект
2.  блок

### Хранение объектов

*   Мы не можем сделать раздел в облачном хранилище. Мы можем хранить только файлы и папки. Это называется хранилищем объектов. Мы не можем установить ОС там, где нет разделов.
*   **пример**
    *   Google drive (Google Compute Engine, GCE), OneDrive, drop box, Microsoft azure.
    *   У Amazon есть своя облачная служба AWS. S3 (Simple Storage Service, SSS) является продуктом amazon, который предоставляет StAAS. Это общественное облако. Любой может использовать свои услуги.
    *   OpenStack - это самое большое частное облако. OpenStack имеет быстрый продукт (хранилище объектов).

### Блокировка блоков

*   Если мы сможем сделать разделы в предоставленном хранилище, то мы сможем установить ОС. У нас есть жесткий диск, и мы можем создавать разделы в них, этот тип хранилища известен как хранилище блоков.
*   **пример**
    *   Служба хранения блоков AWS - EBS (хранилище эластичных блоков)
    *   Облако обеспечивает средство масштабирования хранилища, которое является эластичным облаком.
    *   Блочная память OpenStack - Cinder
