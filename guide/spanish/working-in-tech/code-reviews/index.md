## Revisiones de Código

Las revisiones de código existen para mejorar la calidad del código en un proyecto software y son práctica habitual para muchos equipos de desarrollo.

Las revisiones de código involucran a los miembros del equipo inspeccionando las subidas de código para localizar errores, lógica incorrecta o impulsar potenciales mejoras. Una revisón de código puede ser realizada por parte de los propios  desarrolladores del proyecto en revisión o también por parte de otros desarrolladores de otros equipos de la misma compañía.
A aquellos que no tienen experiencia en esta técnica le puede parecer intimidante. Sin embargo, es un proceso de alto valor para ayudar a crecer como desarrollador. Incluso si una revisión es simplemente una explicación del código a otro programador, el sentimiento de que tu código va a ser leído o inspeccionado por otro programador hace que escribas mejor código en muchos aspectos (métodos más pequeños, comentarios y no ["números mágicos)"](https://en.wikipedia.org/wiki/Magic_number_(programming))).

Relativo a qué código debe ser revisado y cuando, no hay una respuesta clara.
Muchos equipos tienen reglas en las que cada pieza de código que se añade al control de versiones tiene que ser revisada por otro programador, mientras que otros equipos tienen días enteros dedicados a hacer revisiones de código.

Es importante adoptar una actitud humilde durante el proceso de revisión de código. Acepta tus errores, y sé profesional cuando apuntes mejoras o errores en el código de tus compañeros de equipo.
