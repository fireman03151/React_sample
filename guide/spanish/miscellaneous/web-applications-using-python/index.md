Los sitios altamente populares como [YouTube](http://highscalability.com/youtube-architecture) , [Reddit](https://github.com/reddit/reddit) , [Instagram](http://instagram-engineering.tumblr.com/post/13649370142/what-powers-instagram-hundreds-of-instances) , [Quora](http://qr.ae/RUkZJd) , [Pinterest](http://qr.ae/RUkZOJ) , [Dropbox](http://eranki.tumblr.com/post/27076431887/scaling-lessons-learned-at-dropbox-part-1) , [Bitbucket](https://blog.bitbucket.org/2012/08/24/segregating-services/) , etc. están escritos principalmente en Python. Probablemente obtendría una lista más completa investigando por su cuenta.

Obviamente, la lista anterior no es completa; pero esto sería un buen momento para señalar algo a usted - la escritura de su back-end en Python que no se convierte _automáticamente en_ su sitio lo más rápido y escalable como YouTube. Citamos la lista de sitios arriba para señalar simplemente que Python es uno de los favoritos de muchos ingenieros de software en la industria, para el desarrollo web.

Python te da los bloques de lego para construir aplicaciones web robustas y escalables. También puedes disfrutar de la facilidad de escribir módulos de Python mientras lo construyes. Pero un arma es tan potente como su amo. Si no sabes cómo manejarlo, probablemente terminarás disparándote en el pie.
