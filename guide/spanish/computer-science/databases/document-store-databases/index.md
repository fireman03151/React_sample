## Bases de datos del almacén de documentos

Una base de datos de almacén de documentos se considera otro tipo de base de datos NoSQL que es muy similar a una base de datos de valor clave. Un registro en un almacén de documentos representa un único documento estructurado que puede ser diferente de todos los demás documentos en la base de datos. A diferencia de una base de datos relacional, no hay asignación de objetos a varias tablas en la base de datos. En su lugar, podemos tomar todo el objeto y escribirlo directamente en una base de datos del almacén de documentos.

Por ejemplo, podríamos tener el siguiente objeto dentro de nuestro código.

```json
{ 
    "name": "freeCodeCamp", 
    "job": "contributor" 
 } 
```

Este objeto completo se puede escribir directamente en una base de datos de almacén de documentos sin necesidad de análisis adicional.
