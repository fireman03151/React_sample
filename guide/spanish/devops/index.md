## DevOps

![ciclo devops](http://blog.xebialabs.com/wp-content/uploads/2016/03/DevOps-cycle-PPT-COLOURS.png)

DevOps es la integración del trabajo entre las funciones de Desarrollo y Operaciones. Un enfoque para hacer esto es automatizar la forma en que el software se integra y despliega a medida que migra entre los entornos de desarrollo, pruebas y producción. Esto facilita los procesos de instalación del software, lo que reduce idealmente la posibilidad de error.

Alternativamente, el desarrollo de software se puede hacer dentro de silos. One silo es responsable de tomar solicitudes de funciones, luego convertirlas en componentes de software con cualquier lenguaje de programación, y las pruebas básicas se realizan en un entorno con datos ficticios. Cuando se completan las pruebas locales, los productos de trabajo se envían a un equipo de operaciones independiente que es responsable de luego reinstalar el software en un entorno de producción separado que admita el tráfico de usuarios reales. Una vez que el software está en este entorno, los equipos de operaciones mantienen la responsabilidad del código, incluida la identificación de cualquier defecto del producto. Si se encuentran estos defectos, se comparten con el equipo de desarrollo, que intenta resolverlos, luego entrega una nueva versión del software a los equipos de operaciones.
