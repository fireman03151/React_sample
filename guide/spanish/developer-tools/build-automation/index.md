## Construir automatización

Build es una versión del programa de software. El término se usa en referencia a un programa que aún está en desarrollo. La automatización de la compilación es el proceso de automatizar la creación de una compilación de software y los procesos asociados, que incluyen: compilar el código fuente de la computadora en un código binario, empaquetar el código binario y ejecutar pruebas automatizadas. El software de automatización de compilación automatiza las tareas comúnmente realizadas en el proceso de desarrollo de software. La compilación se automatiza cuando los pasos para crear un software son repetibles, no requieren intervención humana directa y se pueden realizar en cualquier momento sin otra información que la que se almacena en el repositorio de control de código fuente. El software de automatización de compilación a menudo se integra con productos de prueba de software, sistemas de control de versiones, IDE, herramientas de seguimiento de errores y proveedores de PaaS. Históricamente, la automatización de la construcción se realizó a través de 'makefiles'. Hoy en día, hay dos categorías generales de herramientas para la automatización de la construcción:

## \- Construir utilidad de automatización:

Su propósito principal es generar artefactos de compilación a través de actividades como compilar y vincular código fuente.

## \- Construir servidores de automatización:

Son herramientas generales basadas en web que ejecutan utilidades de automatización de compilación de forma programada o activada; El servidor de integración continua es un tipo de servidor de automatización de compilación.

Existen tres subcategorías para la automatización de la compilación: integración continua, gestión de la configuración y otra automatización de la compilación.
