你是一个像我一样的露营者，你进行锻炼，你就会被卡住......就像其他人一样。你不知道如何开始，你盯着你的编辑，你认为你一定有问题，当然你不是开发人员的材料......你错了！也许你就像我一样？我是一个视觉人，我喜欢比文字更好的绘画。每次在架构中放置相同的问题对我来说比你给我一个文本更有意义。那么......当我在解决/理解练习时遇到问题，我该怎么办？我开始画画。如果我需要更多将其转换为代码，我的绘图到位后，我也可以用伪代码编写它。之后将其转换为代码应该不是很难。

# 那么这个流程图（图纸）和伪代码是什么？

在我大学的第一个学期，我们有一个关于算法介绍的课程。这是我们第一次了解这些东西的地方。我们了解到使用流程图和伪代码开发了一个好的算法和良好的逻辑编程。

**流程图**表示从上到下的程序流程。每个命令都在此表示。根据命令的性质，您可以使用不同的形状。我主要使用的一些（你可以谷歌更多，谷歌是你的朋友，当你知道谷歌的东西）是：

有关这方面的更多信息，请访问[https://en.wikipedia.org/wiki/Flowchart](https://en.wikipedia.org/wiki/Flowchart) 。

**伪代码**是一种非正式语言，可以帮助开发人员编写算法。它是一个基于文本的设计工具，它使用人类可读的语言。这是一个描述算法的结构化英文文本。

Free Code Camp课程中的每个算法都可以使用伪代码解决，然后在功能性javascript代码中使用javascript进行翻译。
