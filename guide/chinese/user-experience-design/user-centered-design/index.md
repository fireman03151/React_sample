## 以用户为中心的设计

以用户为中心的设计（UCD）是一系列流程，专注于将用户置于产品设计和开发的中心。 UCD是一个 迭代设计方法，旨在通过调查（例如，调查和调查）的混合来开发对用户需求的理解 采访）和生成（例如，头脑风暴）方法和工具。至关重要的是，UCD在所有设计和评估阶段都涉及用户。 设计师检查产品的可用性，特性，环境和任务，并广泛关注设计过程的所有步骤。

UCD的主要目标是围绕用户使用产品的方式优化产品，服务或流程，而不是强迫用户确认或 改变他们的行为以适应产品。

### UCD进程的阶段

1.  指定使用环境：确定将使用该产品的人员，他们将使用该产品的人员以及他们将在何种条件下使用该产品。
2.  指定要求：确定产品成功必须满足的任何业务要​​求或用户目标。
3.  创建设计解决方案：这个过程的一部分可以分阶段完成，从粗略的概念到完整的设计。
4.  评估设计：理想情况下，通过实际用户的可用性测试进行评估，与质量测试一样，是良好的软件开发

### 以用户为中心的设计与以人为本的设计

以用户为中心的设计通常与以人为本的设计互换使用，但前者实际上是后者的一个子集。以用户为中心的设计不仅关注于任何特定的人类用户，而且关注特定类型的用户。因此，以用户为中心的设计需要更深入地分析用户 - 您的目标受众。它不仅关乎一个人的一般特征;它是针对目标用户的特定习惯和偏好，为特定问题提出正确的解决方案。

以用户为中心的设计考虑了年龄，性别，社会地位，教育和专业背景，影响因素，产品使用期望和需求以及许多其他重要因素，这些因素可能因不同的细分市场而异。以用户为中心的设计是对用户的深入研究。习惯，从他们与产品的互动，到他们对产品应该如何和表现的看法。
