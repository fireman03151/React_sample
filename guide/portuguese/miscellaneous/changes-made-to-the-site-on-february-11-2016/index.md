## Corrigindo os gargalos que fizeram o FreeCodeCamp.com tão lento

*   Nossa comunidade de código aberto está crescendo rapidamente e, nesta semana, nosso site diminuiu o ritmo durante os horários de pico.
*   Nós analisamos vários gargalos e os corrigimos. O Free Code Camp agora deve estar mais rápido do que nunca.
*   Um enorme obrigado a [@berkeleytrue](https://gitter.im/berkeleytrue) por liderar este esforço, e aos muitos campistas que ajudaram a controlar essas melhorias.

## Apresentando nosso novo Wiki

*   Nosso wiki está agora no próprio FreeCodeCamp.com (ao contrário do Wiki do GitHub do nosso projeto principal).
*   Nosso wiki aparece como nosso bate-papo e mapa. Congratulamo-nos com comentários sobre esta nova funcionalidade. Teclas de atalho, artigos wiki com reconhecimento de contexto e outros recursos virão em breve.
*   Um grande obrigado a [@saintpeter](https://gitter.im/saintpeter) , [@hallaathrad](https://gitter.im/hallaathrad) e [@awesomeaniruddh](https://gitter.im/awesomeaniruddh) , juntamente com muitos outros campistas por seu trabalho duro nisso!

## Notícias de Retirar Camper

*   No ano passado, o Camper News foi um sucesso modesto. Mas também o nosso subreddit, que permite comentários, moderação e muitos outros recursos. Então estamos mudando para o Reddit para compartilhamento de notícias.
*   Recomendamos criar uma conta do Reddit se você ainda não tiver uma e se [juntar ao nosso subreddit](https://www.reddit.com/r/freecodecamp) . Agora você pode compartilhar links com toda a nossa comunidade, bem como seus pensamentos e perguntas.
*   Manteremos o Camper News ao vivo por um tempo, mas estamos removendo o link para ele do nosso menu de navegação e removendo o desafio que o introduziu.

## Atualizando Desafios

*   Muitos campistas pediram que criassemos um projeto mais fácil, com foco em HTML, para preceder a "Criar uma página da Web de portfólio pessoal". Então, criamos um novo projeto que pode ser concluído usando apenas HTML e Bootstrap chamado "Build a Tribute Page".
*   Também riscalizamos nossos projetos front-end. Veja como eles estão agora estruturados:
    *   Projetos básicos de front end
        *   Construa uma página de homenagem
        *   Criar uma página da Web de portfólio pessoal
    *   Projetos de Front End Intermediários
        *   Construa uma máquina de cotação aleatória
        *   Mostrar o clima local
        *   Construa um visualizador da Wikipedia
        *   Use a API JSON do Twitch.tv
    *   Projetos avançados de front end
        *   Construa uma calculadora JavaScript
        *   Construa um relógio Pomodoro
        *   Construa um jogo Tic Tac Toe
        *   Construa um jogo de Simon
*   Você notará que removemos o projeto "Construa um Visualizador de Notícias da Camper", já que estamos aposentando a Camper News. Se você já concluiu esse desafio anteriormente, a "Página do Build a Tribute" já deve estar marcada como concluída para você no seu mapa.
*   Além disso, para "Construir um jogo Tic Tac Toe", você não precisa mais torná-lo imbatível. (Você certamente pode, se quiser - [eis como](http://neverstopbuilding.com/minimax) . Embora fazer isso seja um exercício útil, escrever uma AI é uma saída do foco do projeto, que é o desenvolvimento do front end.
*   Adicionamos três novos desafios opcionais que ajudam você a se juntar ao nosso:
    *   [subreddit](https://reddit.com/r/freecodecamp)
    *   [Publicação média](https://medium.freecodecamp.com)
    *   [Canal Twitch.tv](https://twitch.tv/freecodecamp)
