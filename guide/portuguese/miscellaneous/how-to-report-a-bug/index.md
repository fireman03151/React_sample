---
title: How to Report a Bug
localeTitle: Como relatar um bug
---
Obrigado por se esforçar para relatar um problema com o freeCodeCamp.

Se você acha que encontrou um bug no freeCodeCamp, siga estas etapas para resolver o problema:

1.  Tente redefinir o código no editor usando o botão de redefinição na página. Isso resolverá a maioria dos problemas se, de alguma forma, você alterar algum código, o que está afetando o desafio de alguma forma. A reinicialização limpa o código para seu estado original como se o desafio tivesse sido apresentado a você pela primeira vez.
    
2.  Se a página parece quebrada de alguma forma, tente fazer um [Hard Refresh](http://forum.freecodecamp.com/t/refresh-your-browser-cache/19365) da página. Isso atualizará qualquer código antigo que possa ter sido armazenado em cache no seu navegador.
    
    *   Se o seu código trava o seu navegador, tente também [Limpar o armazenamento local do seu navegador](http://forum.freecodecamp.com/t/removing-all-locally-stored-challenges/19129) .
3.  [Peça ajuda para as salas de chat](http://forum.freecodecamp.com/t/how-to-get-help-on-gitter/19130) . Geralmente, você pode obter uma resposta rápida de um membro da comunidade útil.
    
4.  [Peça ajuda no fórum](http://forum.freecodecamp.com/) . Ocasionalmente, você pode ver que suas consultas não são abordadas no bate-papo imediatamente ou pode ser algo que precise de uma discussão. Nesse caso, verifique se você está pesquisando neste fórum e, se não conseguir encontrar uma resposta satisfatória, pergunte-a aqui.
    
5.  Se o seu problema tiver confundido todos os usuários nas salas de bate-papo e no fórum, tente [Pesquisar seu problema no GitHub](http://forum.freecodecamp.com/t/searching-for-existing-issues-in-github/18390) para qualquer pessoa que tenha postado sobre um problema semelhante. Se alguém tiver, você pode revalorizar o problema clicando no botão ( ![:thumbsup:](https://forum.freecodecamp.com/images/emoji/emoji_one/thumbsup.png?v=3 ":afirmativo:") ) no post de abertura (primeiro post) no tópico de edição.
    
6.  Se você não encontrar nenhum problema relevante no GitHub, [crie um novo problema](http://forum.freecodecamp.com/t/creating-a-new-github-issue/18392) e nossa equipe de desenvolvimento analisará seu problema.
    

**DISPOSITIVOS MÓVEIS:**  
Alguns desafios podem não funcionar em dispositivos móveis (navegadores móveis), nós tentamos o nosso melhor para apoiá-los, mas recomendamos que você tente verificar o desafio em um navegador de desktop antes de confirmar como um bug.

**IMPORTANTE:**  
Antes de relatar um novo problema, sempre receba uma confirmação de terceiros de alguém nas salas de bate-papo ou no fórum. Lembre-se de que o rastreador de problemas é estritamente para relatar bugs ou melhorias, não é um lugar para procurar ajuda para resolver os desafios.
