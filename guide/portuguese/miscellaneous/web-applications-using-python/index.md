Sites altamente populares como o [YouTube](http://highscalability.com/youtube-architecture) , [Reddit](https://github.com/reddit/reddit) , [Instagram](http://instagram-engineering.tumblr.com/post/13649370142/what-powers-instagram-hundreds-of-instances) , [Quora](http://qr.ae/RUkZJd) , [Pinterest](http://qr.ae/RUkZOJ) , [Dropbox](http://eranki.tumblr.com/post/27076431887/scaling-lessons-learned-at-dropbox-part-1) , [Bitbucket](https://blog.bitbucket.org/2012/08/24/segregating-services/) etc. são escritos principalmente em Python. Você provavelmente obteria uma lista mais abrangente, investigando por conta própria.

Obviamente, a lista acima não é completa; mas esse seria um ótimo momento para apontar algo para você: escrever seu back-end no Python não tornaria o site _automaticamente_ tão rápido e escalonável quanto o YouTube. Citamos a lista de sites acima para simplesmente indicar que o Python é o favorito de muitos engenheiros de software na indústria, para o desenvolvimento da web.

O Python fornece os blocos lego para criar aplicativos da Web robustos e escalonáveis. Você também pode aproveitar a facilidade de escrever módulos Python ao montá-lo. Mas uma arma é tão potente quanto seu mestre. Se você não sabe como manejá-lo, você provavelmente acabaria atirando no próprio pé.
