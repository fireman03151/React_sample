Esta lista foi criada através da compilação de problemas e questões comuns ao trabalhar nos desafios do algoritmo, eles devem ser úteis em geral.

1.  Se você não sabe por onde começar, escreva algum pseudocódigo. Planeje a lógica que você está tentando implementar em seu idioma nativo antes de tentar colocá-lo no código.
    
2.  Comente seu código! Não é possível enfatizar isso o suficiente, se você se perder ou fizer uma pausa, volte e precise saber o que seu código está tentando fazer. Ele também reforçará o que o código significa para você e ajudará outras pessoas a entender o que seu código faz se você ficar preso e pedir ajuda. Se você tem alguma outra dica para os campistas, eu adoraria vê-los!
    
3.  Use o console do depurador javascript chrome em conjunto com algo como o NetBeans IDE ou o texto sublime para escrever seu código. Ambos são gratuitos para download. O editor de código freeCodeCamp é ótimo, mas você pode obter muito mais feedback sobre o comportamento do seu código usando o depurador JS do chrome e um monte de **console.log (myVariableGoesHere);**
