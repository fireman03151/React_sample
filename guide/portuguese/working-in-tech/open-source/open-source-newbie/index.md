localeTitle: A batalha do código aberto

Como alguém novo no desenvolvimento de software, contribuindo para projetos de código aberto, pode parecer intimidador e absolutamente aterrorizante. O desenvolvimento de software de aprendizagem foi uma jornada inesperada que tive o prazer de seguir. Tendo o apoio de meus familiares e amigos tornaram a jornada menos desafiadora, então não foi um caminho fácil. Eu bati muitos obstáculos no caminho, mas continuo me desafiando todos os dias. Li muitos artigos sugerindo que, como desenvolvedor de software júnior, é importante contribuir com fontes abertas. Eu procurei no innerwebs por um projeto open source que eu senti que poderia resolver facilmente.

A dura realidade é que a maioria das fontes abertas lá fora não é tão fácil quanto eu pensava que seria. O GitHub em geral tem sido um desafio para aprender e eu sinto que aprendo algo novo a cada dia, mas olhando para vários projetos open source "amigáveis ​​ao iniciante", eu cheguei à completa conclusão de que vai demorar muito mais tempo do que eu pensado para ser capaz de contribuir com código em projetos de código aberto.

## Lições aprendidas

Mesmo que o parágrafo acima pode parecer um pouco fora de colocar não se preocupe. Sim, os projetos de código aberto não são nada para considerar um negócio sério. Leve o seu tempo, leia a documentação ou leia-me, faça pesquisas e nunca desista ou pare de aprender. Não importa se você é um iniciante ou um desenvolvedor experiente, sempre haverá algo novo para aprender.
