## Revisões de Código

As revisões de código existem para melhorar a qualidade do código num projeto de software e são muito comuns em várias equipes de desenvolvimento de software. As revisões de código envolvem os membros da equipe que investigam os envios de códigos uns dos outros para verificar se há erros, lógica incorreta ou potenciais melhorias. A revisões de código podem ser efetuadas pelos desenvolvedores do projeto sob revisão e, também, por desenvolvedores de outras equipas na mesma empresa. Para aqueles que não experimentaram o processo de revisão de código antes, pode ser intimidante; no entanto, é um processo inestimável para ajudá-lo crescer como um desenvolvedor. Mesmo que a revisão seja apenas uma explicação do código para outro desenvolvedor, a ideia de que o seu código também será revisado/inspecionado por outro desenvolvedor faz com que escreva código melhorado de várias maneiras (métodos mais curtos, comentários e nada de "números mágicos").

No que respeita ao que é revisado e quando o fazer, não existe uma resposta correta. Muitas equipes têm regras que impões que cada novo pedaço de código de uma nova versão tem de ser revisado por outro programador, enquanto outras equipes dedicam dias inteiros em que escrutinam o código uns dos outros.

É importante ser humilde durante todo o processo de revisão de código. Aceite seus erros e seja profissional ao apontar melhorias ou erros no código do seu companheiro membros do time.
