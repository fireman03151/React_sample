## Seguro

SAFe significa "Scaled Agile Framework" e é uma estrutura de desenvolvimento de software ágil projetada para o desenvolvimento ágil de larga escala envolvendo muitas equipes.

O site principal ( [http://www.scaledagileframework.com/](http://www.scaledagileframework.com/) ) é uma base de conhecimento on-line disponível gratuitamente para a implementação do SAFe.

## Princípios

Existem nove princípios básicos para o SAFe

1.  **Adote uma visão econômica** - avalie os custos econômicos de decisões, desenvolvimentos e riscos
2.  **Aplique o pensamento sistêmico** - os processos de desenvolvimento são interações entre os sistemas utilizados pelos trabalhadores e, portanto, precisam visualizar o progresso com o pensamento sistêmico
3.  **Assumir variabilidade; preservar opções** - mudança de requisitos, flexibilidade e uso de dados empíricos para estreitar o foco
4.  **Construa de forma incremental com ciclos de aprendizado rápidos e integrados** - construções rápidas e incrementais permitem um feedback rápido para alterar o projeto, se necessário
5.  **Marcos básicos na avaliação objetiva de sistemas de trabalho** - avaliação objetiva fornece informações importantes, como finanças e governança, como feedback
6.  **Visualize e limite o WIP, reduza o tamanho dos lotes e gerencie os comprimentos das filas** - faça isso para obter um fluxo contínuo para mover-se rapidamente e visualizar o progresso; WIP = trabalho em andamento
7.  **Aplicar cadência, sincronizar com o planejamento entre domínios** - a cadência está definindo as datas em que certos eventos acontecem (por exemplo, liberar semanalmente) e a sincronização garante que todos tenham os mesmos objetivos em mente
8.  **Desbloquear a motivação intrínseca dos trabalhadores do conhecimento** - as pessoas trabalham melhor quando você faz uso de suas motivações pessoais
9.  **Descentralizar a tomada de decisão** - permite uma ação mais rápida, que pode não ser a melhor solução, mas permite uma comunicação mais rápida entre as equipes; tomadas de decisões centralizadas podem ser necessárias para decisões mais estratégicas ou globais

## Configurações

Existem quatro variações do SAFe, variando em complexidade e necessidades do seu projeto:

1.  SAFe essencial
2.  Portfólio SAFe
3.  Solução Grande SAFe
4.  SAFe Completo
