## Desenvolvimento Paralelo

O Parallel Development representa o processo de desenvolvimento separado em várias ramificações, para fornecer um produto versátil com lançamentos estáveis ​​e novos recursos. Em um processo de desenvolvimento de software simples e comum, você tem apenas uma ramificação com correções de erros e melhorias, além de novos recursos. No desenvolvimento paralelo, vários ramos podem coexistir.

Normalmente, o desenvolvimento paralelo contém um ramo principal, "mestre", que é o mais estável e contém correções importantes para o código existente. Na ramificação principal, mais ramificações são criadas para adicionar novos "caminhos" ao código existente. Essas ramificações fornecem novos recursos, mas não incluem correções aplicadas no tempo médio a partir da ramificação principal. Os clientes conhecem essas versões e têm equipamentos especiais ou máquinas de teste para testar os novos recursos. Quando os testes de controle de qualidade são passados, a ramificação lateral pode ser mesclada com a ramificação principal para introduzir novos recursos à versão de lançamento.
