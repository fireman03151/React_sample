## Construir Medir Aprender

O ciclo Contruir-Medir-Aprender (Build-Measure-Learn) é um método usado para criar o produto certo. Cunhado no livro "Lean Startup", de Eric Reis, o ciclo permite uma experimentação rápida, com o único propósito de conseguir o ajuste do mercado. Em outras palavras, é um sistema poderoso para validar suposições relativas a um produto que você se propõe a entregar. Analisando o ciclo, ele consiste nas seguintes partes:

![ciclo-construir-medir-aprender](https://steveblank.files.wordpress.com/2015/05/ideas-build-code-measure.jpg)

### Ideia

Cada ciclo começa com uma ideia que fornecerá valor de negócio a alguns usuários. Essa ideia deve consistir em uma visão de um produto - que o direcionará no que construir e uma métrica que apontará se suas suposições sobre o valor do negócio estavam corretas.

### Construir

Para validar a sua ideia, você planeja construir um Produto Mínimo Viável (MVP), combinado com métricas predefinidas (preferencialmente uma), cuja finalidade é validar sua teoria e ajudá-lo a decidir se você deve preservar ou pivotar.

### Medir

Este estágio é focado na coleta de dados e métricas do MVP.

### Aprender

Em seguida, usando os dados coletados, uma decisão precisa ser tomada: se o seu produto é ou não usado pelos usuários e, portanto, você deve preservar ou se os usuários não estão interessados ​​no produto e, como tal, você deve pivotar. A fase de aprendizado, portanto, termina com uma ideia (seja de como expandir o produto ou de como pivotar a partir do produto original), aplicável ao próximo ciclo Construir-Medir-Aprender.
