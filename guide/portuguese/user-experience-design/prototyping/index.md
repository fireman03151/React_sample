## Prototipagem

No desenvolvimento de software, um protótipo é um modelo de trabalho rudimentar de um produto ou sistema de informação, geralmente construído para fins de demonstração ou como parte do processo de desenvolvimento. No modelo de prototipagem do ciclo de vida de desenvolvimento de sistemas (SDLC), uma versão básica do sistema é construída, testada e depois retrabalhada conforme necessário até que um protótipo aceitável seja finalmente alcançado a partir do qual o sistema completo ou produto pode agora ser desenvolvido.

Prototipagem refere-se a um estágio inicial de uma liberação de software na qual a evolução do desenvolvimento e as correções do produto podem ocorrer antes que uma liberação maior seja iniciada. Esses tipos de atividades também podem ser chamadas de fase beta ou teste beta, onde um projeto inicial é avaliado por uma classe menor de usuários antes do desenvolvimento completo.

A prototipagem, bem como testes de amplo espectro e vários lançamentos de software, é parte de um processo mais detalhado produzindo produtos e serviços de software sofisticados. A ideia essencial é que, mesmo quando os recursos de código estejam completos em um projeto, o software, que ainda está em desenvolvimento, pode ter muitos bugs e problemas de usuário. Para muitos deles, ser passado a ferro Além disso, ajuda se o software estiver realmente em uso, mas os desenvolvedores enfrentam a questão de liberar um produto que os usuários finais podem ver como essencialmente falho. Liberar o produto para uma comunidade menor ou restringir seu desenvolvimento em etapas pode ser uma solução muito eficaz.

Em alguns casos, a prototipagem pode envolver voluntários, enquanto em outros casos, clientes especiais ou outros com status especial podem estar envolvido na prototipagem. As equipes de desenvolvimento e as empresas de tecnologia geralmente obtêm feedback sobre um produto durante prototipagem para corrigir problemas antes da distribuição final.
