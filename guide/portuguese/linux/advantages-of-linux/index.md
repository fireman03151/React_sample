## Melhorias constantes e eficientes.

As atualizações do Linux acontecem através da colaboração global de desenvolvedores. Os bugs são documentados e resolvidos muito mais rapidamente com esse tipo de suporte. Além disso, como os desenvolvedores também são os usuários finais, eles têm a motivação adequada para garantir que atenda às necessidades do usuário e que ele seja projetado para funcionar bem.

## Menos risco.

O software é tão bom quanto o suporte dado. Imagine isso: um software é criado pela empresa XYZ e, mais tarde, vai à falência ou é comprado por outra empresa que decide cortar seu apoio. O software nunca veria melhorias ou correções e, portanto, sua utilidade iria inevitavelmente corroer e morrer. O Linux pertence e é operado por nenhuma entidade ou empresa, portanto, essa situação não pode acontecer. Mais do que isso, qualquer um é livre para buscá-lo e contribuir também. O risco de perder suporte para o Linux é muito improvável devido à sua enorme popularidade e uso.

## Leve.

Os requisitos do Linux para rodar em um sistema são muito menores que os do Windows ou Mac. Com a distribuição certa do Linux, um usuário pode ter uma configuração modesta e o Linux fornecerá o valor do sistema. O espaço em disco e a pegada de memória também podem ser menores. Algumas distribuições são adequadas para CPUs que remontam à família Pentium, outras possuem um requisito de apenas 128 MB de RAM e aproximadamente a mesma quantidade de espaço em disco!

## Muito documentado para iniciantes e usuários avançados.

O Linux tem uma comunidade ativa de pessoas dispostas a compartilhar seus conhecimentos e ajuda (muito parecido com o FreeCodeCamp!). Existem utilitários de linha de comando embutidos no Linux que fornecem documentação sobre comandos, bibliotecas, padrões, etc. ( [Man pages](https://en.wikipedia.org/wiki/Man_page) e [Info pages](https://en.wikipedia.org/wiki/Info_(Unix) ), e há documentação disponível na Internet em vários formatos, incluindo [The Linux Documentation Project](http://www.tldp.org/) , [LinuxQuestions](http://www.linuxquestions.org/) , [ServerFault](http://serverfault.com/) e [The Arch Wiki](https://wiki.archlinux.org/) .

Além da documentação, há muitas comunidades úteis e acolhedoras para os recém-chegados fazerem perguntas como [Ask](https://askubuntu.com/) [rss e linuxquestions do](https://www.reddit.com/r/linuxquestions/) [Ubuntu](https://askubuntu.com/) e do [Reddit](https://www.reddit.com/r/linuxquestions/) .

Existem certificações individualizadas que podem ser tomadas e reconhecidas no setor de TI ( [testes](https://www.lpi.org) [Linux +](https://certification.comptia.org/certifications/linux) e [LPIC LPI da](https://www.lpi.org) [CompTIA](https://certification.comptia.org/certifications/linux) ). Embora não seja obrigatório, o aprendizado [C](https://en.wikipedia.org/wiki/C_(programming_language) ) permite que um usuário revise o código do Linux para ver o que está executando.

## Suporte de negócios.

O mundo depende do Linux suportar sistemas críticos para que a demanda não desapareça. Isso é importante não apenas para aqueles que ajudam a contribuir para o Linux, mas também para aqueles que o apóiam (Jobs!). O suporte ao Linux se torna mais crítico em TI, mas também ter esse conhecimento como desenvolvedor os tornará mais arredondados e úteis (ou seja, [desenvolvedores de pilha completa](https://medium.com/chris-messina/the-full-stack-employee-ed0db089f0a1#.ubttrv255) ).

## Interoperabilidade com outros sistemas operacionais.

O Linux possui suporte a drivers para sistemas de arquivos NTFS e HFS + (usados ​​pelo Windows e Macs) e também ao [Samba](https://www.samba.org/) para suporte a arquivos / serviços de impressão em máquinas Windows.

## Melhor suporte de hardware.

Para outros SOs, geralmente um usuário teria que ir ao site do fabricante para obter suporte de driver para diferentes tipos de hardware. O kernel do Linux suporta a maioria do hardware automaticamente via plug-and-play (em grande parte devido à comunidade de código aberto). Alguns fabricantes também desenvolvem versões Linux de seus drivers proprietários que podem ser facilmente instalados através do repositório de software de uma distribuição ou pela instalação manual dos binários fornecidos.

## Disponibilidade de Software / Compatibilidade.

O Linux possui um rico conjunto de aplicativos que estão disponíveis. Tem [aplicativos de escritório](https://www.libreoffice.org/discover/libreoffice/) , navegadores da Web ( [Google Chrome](https://www.google.com/chrome/browser/desktop/) e [Mozilla Firefox](https://www.mozilla.org/en-US/firefox/new/) ), [reprodutores de mídia](http://www.videolan.org/vlc/) , [processamento de](http://www.openshot.org/) [imagem](https://www.gimp.org/) / [vídeo](http://www.openshot.org/) etc. Para jogos de vídeo, o [Steam](http://store.steampowered.com/about/) é executado no Linux, que possui muitos jogos compatíveis. Aplicativos do Windows também podem rodar no Linux (veja o [Wine](https://www.winehq.org/) para detalhes).

## Construído para desenvolvimento.

Como o Linux foi feito originalmente para desenvolvedores por desenvolvedores, eles gastaram muito tempo e esforço aperfeiçoando as ferramentas que eles usariam. Ele tem um poderoso [shell](https://en.wikipedia.org/wiki/Unix_shell) que pode ser usado para uma variedade de tarefas de programação e administrativas ( [Bash](https://en.wikipedia.org/wiki/Bash_(Unix_shell) ) é a opção mais popular e padrão para o Linux). Embora o Linux tenha um equivalente "notepad" no [gedit](https://en.wikipedia.org/wiki/Gedit) , ele também fornece editores baseados em texto mais poderosos e personalizáveis, como o [Vim](https://en.wikipedia.org/wiki/Vim_(text_editor)) ) e o [Emacs](https://en.wikipedia.org/wiki/Emacs) (recomenda-se que os usuários do Linux conheçam pelo menos um desses editores). Há também IDEs disponíveis para desenvolvimento da web, como [Atom](https://atom.io/) , [Aptana](http://www.aptana.com/) / [Eclipse](https://eclipse.org/) , [Sublime](https://www.sublimetext.com/) , [KomodoIDE](http://komodoide.com/) , para citar alguns. O Linux também pode utilizar software como o [Apache](https://httpd.apache.org/) para configurar um servidor web local para testes, o [Git](https://git-scm.com/) para controle de versão e outras ferramentas / linguagens como o [Node.js](https://nodejs.org/en/) / [Ruby](https://www.ruby-lang.org/en/) / [SaSS](http://sass-lang.com/) / [Heroku,](https://www.heroku.com/) todos suportando Linux e ferramentas de linha de comando que podem ser usado em vez de uma GUI.

## Customizável.

O Linux permite que o usuário altere o design e os temas da área de trabalho, adicione widgets e muito mais. Essas alterações podem ser feitas com ambientes de área de trabalho. Ambientes diferentes têm diferentes conjuntos de opções. Alguns populares são: KDE, GNOME, XFCE, Pantheon.
