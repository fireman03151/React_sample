## Engenharia de rede

Um **pacote** é uma abstração de dados transmitidos na _camada_ de _rede_ do modelo OSI. Ele consiste nos dados reais junto com outros campos necessários para sua mediação de um nó para outro na rede. Como a transferência de todos os dados juntos através de uma rede de uma só vez é ineficiente, é dividida em pequenos pacotes e depois repassada.

Um pacote se parece com isso:

![pacote](https://www.computerhope.com/jargon/p/packet.jpg)

A camada de rede divide os dados em pequenos pacotes, cada um com seu identificador exclusivo. Eles são então passados ​​para a Camada de Transporte, que os transporta para o destino através de vários nós intermediários, como roteadores e switches, usando o _Protocolo de Controle de Transmissão_ (TCP) e _o Protocolo de Internet_ (IP). O TCP lida com o roteamento real, enquanto o IP ajuda no endereçamento lógico dos dispositivos (endereço IP) para manter um controle de qual nó enviar o pacote ao lado.

Quando os pacotes chegam ao destino, uma confirmação é enviada de volta. Se, por algum motivo, um pacote for descartado, ele poderá ser requisitado novamente a partir do nó de origem.

Os engenheiros de rede são responsáveis ​​por implementar, manter, apoiar, desenvolver e, em alguns casos, projetar redes de comunicação dentro de uma organização ou entre organizações.

Seu objetivo é garantir a integridade da infraestrutura de rede de alta disponibilidade para fornecer o máximo desempenho para seus usuários. Os usuários podem ser funcionários, clientes, clientes e fornecedores. As redes podem incluir:

computador; voz; firewall. Os engenheiros de rede podem trabalhar internamente como parte da equipe de suporte de TI de uma organização, ou externamente como parte de uma empresa terceirizada de consultoria de redes de TI que trabalha com vários clientes.
