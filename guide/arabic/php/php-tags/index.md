---
title: PHP tags
localeTitle: علامات PHP
---
عندما يقوم PHP بتوزيع ملف ، فإنه يبحث عن علامات فتح وإغلاق ، والتي هي `<?php` و `?>` والتي تخبر PHP لبدء وتفسير تفسير الشفرة بينهما. يسمح التحليل بهذه الطريقة بتضمين PHP في جميع أنواع المستندات المختلفة ، حيث يتم تجاهل كل شيء خارج زوج من علامات الفتح والإغلاق من قبل محلل PHP.

PHP يسمح أيضا لعلامة مفتوحة قصيرة `<?` (وهو أمر غير `short_open_tag php.ini` لأنه متاح فقط في حالة تمكينه باستخدام `short_open_tag php.ini` تكوين `short_open_tag php.ini` ، أو إذا كان PHP قد تم تكوينه باستخدام خيار `--enable-short-tags` ).

إذا كان الملف عبارة عن كود PHP خالص ، فمن الأفضل حذف علامة إغلاق PHP في نهاية الملف. يؤدي ذلك إلى منع المساحات البيضاء أو الخطوط الجديدة التي تتم إضافتها بعد علامة إغلاق PHP ، والتي قد تتسبب في حدوث تأثيرات غير مرغوب فيها لأن PHP سيبدأ التخزين المؤقت للإخراج عندما لا يكون هناك نية من المبرمج لإرسال أي إخراج عند هذه النقطة في البرنامج النصي.

```php
<?php
echo "Hello world";

// ... more code

echo "Last statement";

// the script ends here with no PHP closing tag
```
